etcd-operator

能力

  • 创建/销毁：用户只需最小限度地指定集群的大小，而不是为每个 etcd 成员指定繁琐的设置。
  • 调整大小：用户只需要按规范修改大小，etcd Operator 将负责部署，销毁或重新配置集群成员，例如，从 3 到 5，或从 5 到 3。
  • 备份：etcd Operator 自动并透明地执行备份。用户只需指定备份策略，例如每 30 分钟备份一次，并保留最后 3 次备份。
  • 升级：在不停机地情况下升级 etcd 是一项重要而艰巨地任务。使用 etcd Operator 不仅可以简化操作，还可以避免常见地升级缺陷和错误。

创建/销毁

安装

1.设置RBAC规则，因为etcd operator需要访问Kubernetes 的 Apiserver 来创建对象。

1

$ example/rbac/create_role.sh

上述脚本为 Etcd Operator 定义了如下所示的权限：

  • 对 Pod、Service、PVC、Deployment、Secret 等 API 对象，有所有权限；
  • 对 CRD 对象，有所有权限；
  • 对属于 etcd.database.coreos.com 这个 API Group 的 CR（Custom Resource）对象，有所有权限。

3.安装etcd-cluster deployments。

1

$ kubectl apply -f example/example-etcd-cluster.yaml

以下是 example/example-etcd-cluster.yaml 的具体内容：

apiVersion: "etcd.database.coreos.com/v1beta2"
kind: "EtcdCluster"
metadata:
  name: "example-etcd-cluster"
  ## Adding this annotation make this cluster managed by clusterwide operators
  ## namespaced operators ignore it
  # annotations:
  #   etcd.database.coreos.com/scope: clusterwide
spec:
  size: 3
  version: "3.2.13"

这时会创建一个版本为3.2.13的3节点etcd集群。

4.总结一下：一个由operator控制的etcd集群包括以下资源：（可选，创建一个serviceAccount）

类型 资源 描述
ClusterRole etcd-operator etcd operator规则
ClusterRoleBinding etcd-operator 绑定serviceAccount和clusterRole
Deployment etcd-operator etcd集群operator
CustomResourceDefinition etcdclusters.etcd.database.coreos.com 由etcd-operator创建的EtcdCluster资源
EtcdCluster example-etcd-cluster etcd集群资源

卸载

官方建议在uninstall之前，先删除相关的用户自定义（CR）资源。

1
2
3
4
5

kubectl delete -f example/deployment.yaml
kubectl delete endpoints etcd-operator
kubectl delete crd etcdclusters.etcd.database.coreos.com
kubectl delete clusterrole etcd-operator
kubectl delete clusterrolebinding etcd-operator

故障恢复

Case1: etcd cluster单点故障：由etcd-operator维护故障恢复，无需用户关心。

Case2: etcd-operator故障：由于etcd-operator为deployment资源，会自动拉起新的pod。不会影响etcd集群。

Case3: etcd-operator Deployment被删除：会重新接管EtcdCluster资源，按照配置维护集群状态。不会影响etcd集群。

备份恢复

备份模块：etcd backup operator

恢复模块：etcd restore operator
